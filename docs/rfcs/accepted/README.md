# Accepted RFCs

Accepted RFCs are the deeper reference layer that should back current
implementation. If an accepted RFC diverges from code, trust the code first and
then update the RFC.

## What belongs here

- stable contracts
- accepted architecture decisions
- security and isolation policy
- CLI, capsule, lifecycle, and IPC behavior

This directory is the long-lived technical reference. Topic pages under
`docs/` are the shorter, implementation-first entry points.
